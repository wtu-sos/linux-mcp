## 新增需求

### 需求：回滚文件操作
系统必须支持通过从回收站恢复来回滚文件操作。

#### 场景：回滚 delete_file
- **当** 对 `delete_file` 操作调用 `rollback_operation`
- **则** 系统从回收站将文件恢复到原始路径

#### 场景：回滚 write_file
- **当** 对创建了备份的 `write_file` 操作调用 `rollback_operation`
- **则** 系统从回收站恢复之前的文件版本

#### 场景：回滚 upload_file
- **当** 对创建了备份的 `upload_file` 操作调用 `rollback_operation`
- **则** 系统从回收站恢复之前的文件并删除上传的文件

### 需求：回滚命令执行
系统必须支持使用存储的回滚命令来回滚命令执行操作。

#### 场景：有存储回滚命令时回滚
- **当** 对有 `rollback_command` 的 `run_command` 操作调用 `rollback_operation`
- **则** 系统执行存储的回滚命令并返回结果

#### 场景：无存储回滚命令时回滚
- **当** 对没有 `rollback_command` 的 `run_command` 操作调用 `rollback_operation`
- **则** 系统返回错误，提示此操作没有可用的回滚方案

### 需求：拒绝不可逆操作的回滚
系统必须拒绝对标记为不可逆操作的回滚尝试。

#### 场景：回滚 kill_process
- **当** 对 `kill_process` 操作调用 `rollback_operation`
- **则** 系统返回错误："此操作不可逆，无法回滚"

### 需求：按操作 ID 回滚
系统必须使用审计日志条目 ID 来标识要回滚的操作。

#### 场景：按审计条目回滚
- **当** 使用审计日志条目 ID 调用 `rollback_operation`
- **则** 系统在审计日志中查找操作，确定回滚策略并执行

#### 场景：回滚不存在的操作
- **当** 指定的操作 ID 在审计日志中不存在
- **则** 系统返回操作未找到的错误

### 需求：回滚审计追踪
系统必须在审计日志中记录回滚操作。

#### 场景：回滚被记录
- **当** 执行回滚操作
- **则** 系统追加一条审计条目，操作类型为 "rollback"，引用原始操作 ID
