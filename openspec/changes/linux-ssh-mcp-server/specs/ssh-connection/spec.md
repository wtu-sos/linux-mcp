## 新增需求

### 需求：SSH 连接建立
系统必须使用配置的主机、端口、用户名和认证方式（SSH 密钥或密码）建立到远程 Linux 主机的 SSH 连接。

#### 场景：使用 SSH 密钥连接
- **当** 配置了 SSH 密钥路径且密钥文件存在
- **则** 系统使用 SSH 密钥认证并建立连接

#### 场景：使用密码连接
- **当** 配置了密码且未提供 SSH 密钥路径
- **则** 系统使用密码认证并建立连接

#### 场景：连接超时
- **当** 远程主机在配置的超时时间内（默认 30 秒）无法连接
- **则** 系统返回明确的连接超时错误信息

#### 场景：认证失败
- **当** 凭据无效
- **则** 系统返回认证失败的错误信息，不暴露敏感细节

### 需求：连接生命周期管理
系统必须管理 SSH 连接的生命周期，包括连接、断开和自动重连。

#### 场景：延迟连接
- **当** 首次调用 MCP 工具且没有活跃连接
- **则** 系统在执行工具前自动建立 SSH 连接

#### 场景：失败自动重连
- **当** 工具操作期间现有 SSH 连接断开
- **则** 系统尝试重连一次并重试操作

#### 场景：显式断开
- **当** MCP 服务器关闭
- **则** 系统优雅地关闭 SSH 连接

### 需求：配置加载
系统必须从 JSON 配置文件和环境变量加载配置，环境变量优先级更高。

#### 场景：从配置文件加载
- **当** 配置路径下存在有效的 JSON 配置文件
- **则** 系统从文件中读取主机、端口、用户名、超时和认证类型

#### 场景：环境变量覆盖
- **当** 配置文件和环境变量都指定了同一设置
- **则** 环境变量的值优先

#### 场景：缺少必要配置
- **当** 缺少必要配置（主机、用户名）
- **则** 系统报告缺少哪些必要字段并以明确错误退出
