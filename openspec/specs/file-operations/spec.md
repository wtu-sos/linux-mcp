## 新增需求

### 需求：读取远程文件
系统必须从远程 Linux 主机读取文件内容。

#### 场景：读取文本文件
- **当** 提供有效的文件路径（例如 `/etc/hostname`）
- **则** 系统以文本形式返回文件内容

#### 场景：读取不存在的文件
- **当** 指定的文件路径不存在
- **则** 系统返回文件未找到的错误

#### 场景：读取文件超出大小限制
- **当** 文件大小超过配置的最大读取大小（默认 10MB）
- **则** 系统返回文件过大的错误

### 需求：写入远程文件并备份
系统必须将内容写入远程 Linux 主机上的文件，如果文件已存在则备份旧版本到回收站。

#### 场景：创建新文件
- **当** 提供了新文件路径和内容且文件不存在
- **则** 系统在远程主机上创建包含指定内容的文件

#### 场景：覆盖已有文件并备份
- **当** 提供了已有文件路径和新内容
- **则** 系统先将旧文件移动到 `~/.linux_mcp_trash/<时间戳>/`，再写入新内容

#### 场景：写入无权限目录
- **当** 目标路径所在目录用户没有写权限
- **则** 系统返回权限拒绝错误

### 需求：列出目录
系统必须列出远程主机上指定路径的文件和目录。

#### 场景：列出目录内容
- **当** 提供有效的目录路径（例如 `/var/log`）
- **则** 系统返回条目列表，包含名称、类型（文件/目录/符号链接）、大小和修改时间

#### 场景：列出不存在的目录
- **当** 指定路径不存在或不是目录
- **则** 系统返回相应错误

### 需求：删除文件到回收站
系统必须将删除的文件移动到回收站目录，而非永久删除。

#### 场景：删除已有文件到回收站
- **当** 提供有效的文件路径
- **则** 系统将文件移动到 `~/.linux_mcp_trash/<时间戳>/`，保留原始文件名，在 `.index.json` 中记录条目，并返回成功及回收站条目 ID

#### 场景：删除不存在的文件
- **当** 指定文件不存在
- **则** 系统返回文件未找到的错误

#### 场景：删除目录到回收站
- **当** 提供目录路径
- **则** 系统将整个目录移动到回收站，保留其结构

### 需求：上传文件到远程主机并防覆盖
系统必须通过 SFTP 将本地文件上传到远程 Linux 主机，如目标已存在文件则备份。

#### 场景：上传文件
- **当** 提供本地文件路径和远程目标路径且目标不存在文件
- **则** 系统将文件上传到远程主机并返回成功

#### 场景：上传文件覆盖已有文件
- **当** 远程目标路径已存在文件
- **则** 系统先将已有文件备份到回收站，再上传新文件

#### 场景：上传文件超出大小限制
- **当** 本地文件超过配置的最大上传大小（默认 10MB）
- **则** 系统返回文件过大的错误

### 需求：从远程主机下载文件
系统必须通过 SFTP 从远程 Linux 主机下载文件到本地路径。

#### 场景：下载文件
- **当** 提供远程文件路径和本地目标路径
- **则** 系统将文件下载到本地路径并返回成功
