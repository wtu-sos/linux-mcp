## 新增需求

### 需求：列出运行中的进程
系统必须列出远程 Linux 主机上运行中的进程。

#### 场景：列出所有进程
- **当** 未指定过滤器
- **则** 系统返回进程列表，包含 PID、用户、CPU 占用率、内存占用率和命令

#### 场景：按名称过滤
- **当** 提供了进程名称过滤器（例如 `nginx`）
- **则** 系统仅返回匹配该名称的进程

#### 场景：结果数量限制
- **当** 进程数量超过配置的限制（默认 200）
- **则** 系统返回按 CPU 使用率排序的前若干进程及截断提示

### 需求：按 PID 终止进程并确认
系统必须按 PID 终止远程主机上的进程，由于此操作不可逆，需要显式确认。

#### 场景：确认后终止已有进程
- **当** 提供有效 PID 且 `confirm_destructive` 设为 true
- **则** 系统向进程发送 SIGTERM 并返回成功

#### 场景：未确认时终止进程
- **当** 提供有效 PID 但 `confirm_destructive` 未设为 true
- **则** 系统拒绝操作并返回错误："此操作不可逆。请设置 confirm_destructive=true 以继续。"

#### 场景：终止不存在的 PID
- **当** 指定 PID 不存在
- **则** 系统返回进程不存在的错误

#### 场景：确认后强制终止
- **当** force 标志设为 true 且 `confirm_destructive` 为 true
- **则** 系统发送 SIGKILL 而非 SIGTERM

### 需求：按名称终止进程并确认
系统必须按名称终止远程主机上匹配的进程，需要显式确认。

#### 场景：确认后按名称终止进程
- **当** 提供进程名称（例如 `nginx`）且 `confirm_destructive` 为 true
- **则** 系统向所有匹配该名称的进程发送 SIGTERM 并返回终止数量

#### 场景：未确认时按名称终止进程
- **当** 提供进程名称但 `confirm_destructive` 不为 true
- **则** 系统拒绝操作并返回需要确认的错误

#### 场景：没有匹配的进程
- **当** 没有进程匹配给定名称
- **则** 系统返回未找到进程的提示
