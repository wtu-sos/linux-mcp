## 新增需求

### 需求：执行 Shell 命令
系统必须在远程 Linux 主机上执行 Shell 命令，并返回 stdout、stderr 和退出码。

#### 场景：命令执行成功
- **当** 提供有效的命令字符串（例如 `ls -la /tmp`）
- **则** 系统返回 stdout 内容、空的 stderr 和退出码 0

#### 场景：命令执行出错
- **当** 命令执行失败（例如 `cat /nonexistent`）
- **则** 系统返回包含错误信息的 stderr 和非零退出码

#### 场景：命令超时
- **当** 命令超过配置的执行超时时间（默认 60 秒）
- **则** 系统终止命令并返回超时错误

#### 场景：指定工作目录
- **当** 为命令指定了工作目录
- **则** 系统在该目录中执行命令

#### 场景：设置环境变量
- **当** 命令附带了额外的环境变量
- **则** 系统为命令执行设置这些变量

### 需求：命令输出大小限制
系统必须对命令执行结果强制实施最大输出大小限制。

#### 场景：输出超出限制
- **当** 命令 stdout 超过配置的最大输出大小（默认 1MB）
- **则** 系统截断输出并附加截断提示

### 需求：命令危险度分类
系统必须在执行前将命令分类到安全等级。

#### 场景：检测到安全命令
- **当** 命令匹配只读模式（例如 `ls`、`cat`、`grep`、`df`、`ps`）
- **则** 系统直接执行命令，不要求提供回滚命令

#### 场景：危险命令有回滚方案
- **当** 命令匹配危险模式（例如 `rm`、`mv`、`dd`、`apt remove`）且提供了回滚命令
- **则** 系统执行命令并存储回滚命令以备撤销

#### 场景：危险命令无回滚方案
- **当** 命令匹配危险模式且未提供回滚命令
- **则** 系统拒绝执行并返回错误，提示需要提供回滚命令

#### 场景：路径感知风险升级
- **当** 通常安全的命令操作高风险路径（例如 `cat > /etc/passwd`）
- **则** 系统因路径风险加权将其归类为危险命令

### 需求：回滚命令参数
系统必须接受命令执行的可选 `rollback_command` 参数。

#### 场景：提供了回滚命令
- **当** 命令执行时带有 `rollback_command` 参数
- **则** 系统将回滚命令存储在审计日志中，使其可用于回滚执行

#### 场景：执行回滚命令
- **当** 对之前执行的命令调用 `rollback_operation` 工具
- **则** 系统执行存储的回滚命令并记录结果
