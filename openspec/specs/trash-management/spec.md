## 新增需求

### 需求：回收站目录结构
系统必须在远程主机上维护 `~/.linux_mcp_trash/` 回收站目录及 JSON 索引文件。

#### 场景：自动创建回收站目录
- **当** 首次将文件移入回收站且回收站目录不存在
- **则** 系统创建 `~/.linux_mcp_trash/` 并初始化 `.index.json`，包含空的 entries 数组

#### 场景：回收站条目结构
- **当** 文件移入回收站
- **则** 系统创建带时间戳的子目录（格式：`YYYYMMDD_HHmmss_<随机6位>`）并将文件移入，保留原始文件名

### 需求：回收站索引管理
系统必须在回收站目录中维护 `.index.json` 文件，追踪所有条目。

#### 场景：创建索引条目
- **当** 文件移入回收站
- **则** 系统向 `.index.json` 追加条目，包含：`id`、`original_path`、`operation`（delete_file/write_file_backup/upload_backup）、`timestamp`、`rolled_back`（false）

#### 场景：恢复时更新索引条目
- **当** 从回收站恢复文件
- **则** 系统将 `rolled_back` 设为 true 并记录 `restored_at` 时间戳

### 需求：列出回收站内容
系统必须提供列出回收站目录内容的工具。

#### 场景：列出所有回收站条目
- **当** 调用 `list_trash`
- **则** 系统返回 `.index.json` 中所有条目及其元数据

#### 场景：列出未回滚的条目
- **当** 调用 `list_trash` 并设置 `include_rolled_back: false`
- **则** 系统仅返回 `rolled_back` 为 false 的条目

### 需求：从回收站恢复文件
系统必须将文件从回收站恢复到原始位置。

#### 场景：恢复已删除文件
- **当** 调用 `restore_file` 并指定有效的回收站条目 ID
- **则** 系统将文件从回收站子目录移回其 `original_path`，并将条目标记为已回滚

#### 场景：恢复时路径冲突
- **当** `original_path` 已存在文件
- **则** 系统返回冲突错误并建议使用其他目标路径

#### 场景：恢复不存在的条目
- **当** 指定的回收站条目 ID 不存在
- **则** 系统返回条目未找到的错误

### 需求：清空回收站
系统必须提供永久删除回收站所有内容的工具。

#### 场景：确认后清空回收站
- **当** 调用 `empty_trash` 并设置 `confirm_destructive: true`
- **则** 系统删除所有回收站子目录并将 `.index.json` 重置为空

#### 场景：未确认时清空回收站
- **当** 调用 `empty_trash` 但未设置 `confirm_destructive: true`
- **则** 系统拒绝操作并返回需要确认的错误

### 需求：回收站大小限制
系统必须在回收站目录超过可配置的大小阈值时发出警告。

#### 场景：回收站超出警告阈值
- **当** 回收站目录总大小超过配置的警告阈值（默认 100MB）
- **则** 系统在响应中包含警告，建议通过 `empty_trash` 清理
